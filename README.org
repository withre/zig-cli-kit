** 🌄 Overview

*zig-cli-kit* defines command-line interfaces as declarative data and hands
them to =App.run=. It targets the Zig =0.17= development line and keeps Zig
=0.16.0= green as the deployment baseline. The API is built around =std.Io=
/ =std.process.Init=; nothing here predates Zig 0.16.

The library never calls =std.process.exit= or =std.debug.print= — all
output goes through writers the caller supplies, and parse failures come
back as narrow =Error= values. The caller owns argv, env, writers, and the
exit code.

It stays deliberately small: no framework runtime, no dependencies beyond
the Zig stdlib, no generated command layer, no ambition to become Zig's
=clap=.

** ✨ Features

- Declarative =App= / =Command= / =FlagDef= / =ArgDef= definitions
- Nested subcommands and aliases
- Global flags that work before or after commands
- Environment binding with =flag > env > default= priority
- Required-flag validation and mutual-exclusion checks
- =--= rest-argument capture
- Did-you-mean suggestions for commands and flags (recurses into subcommands)
- Negatable boolean flags (=--no-X=)
- TTY-aware coloured help output

** 🚀 Quick Start

Define the command tree as data; handlers receive an =Allocator= and a
=*cli.Context= they read flags, positionals, env, and writers from.

#+begin_src zig
  const std = @import("std");
  const cli = @import("zig-cli-kit");

  fn handleStatus(allocator: std.mem.Allocator, ctx: *cli.Context) !void {
      _ = allocator;
      try ctx.stdout.print("status: project={s}\n", .{ctx.flag("project")});
  }

  const app: cli.App = .{
      .name = "example",
      .description = "example CLI",
      .global_flags = &.{
          .{ .name = "project", .short = 'p', .default = "default", .env = "PROJECT", .description = "Project name" },
      },
      .commands = &.{
          .{ .name = "status", .description = "Show status", .run = handleStatus },
      },
  };
#+end_src

=App.run= never exits the process and never touches global stdio; it uses
the writers you pass. Method receiver first, then stdlib interface values
(allocator, IO, writers, env block), then data:

#+begin_src zig
  try app.run(
      allocator,         // std.mem.Allocator
      io,                // std.Io
      stdout_writer,     // *std.Io.Writer
      stderr_writer,     // *std.Io.Writer
      env_block,         // [*:null]const ?[*:0]const u8
      args,              // []const []const u8
  );
#+end_src

The Zig 0.16+ "juicy =main=" entry point passes a pre-initialised
=std.process.Init= with allocator, IO, args, and the environment block.
The library consumes it directly, and this stays the target shape for 0.17.

#+begin_src zig
  const std = @import("std");
  const cli = @import("zig-cli-kit");

  fn handleStatus(allocator: std.mem.Allocator, ctx: *cli.Context) !void {
      _ = allocator;
      try ctx.stdout.print("status: project={s}\n", .{ctx.flag("project")});
  }

  const app: cli.App = .{
      .name = "hello",
      .description = "hello example for zig-cli-kit",
      .global_flags = &.{
          .{ .name = "project", .short = 'p', .default = "default", .env = "PROJECT", .description = "Project name" },
      },
      .commands = &.{
          .{ .name = "status", .description = "Show status", .run = handleStatus },
      },
  };

  pub fn main(init: std.process.Init) !void {
      // Buffered writers for stdout / stderr. The library never touches
      // global stdio, so the caller owns the buffers and the flush.
      var stdout_buf: [4096]u8 = undefined;
      var stderr_buf: [4096]u8 = undefined;
      var stdout = std.Io.File.stdout().writer(init.io, &stdout_buf);
      var stderr = std.Io.File.stderr().writer(init.io, &stderr_buf);
      defer stdout.interface.flush() catch {};
      defer stderr.interface.flush() catch {};

      // Materialise the argv iterator into the `[]const []const u8` the
      // library expects. Zig's iterator handles WASI/Windows quirks.
      var args_iter = try std.process.Args.Iterator.initAllocator(init.minimal.args, init.gpa);
      defer args_iter.deinit();
      var args: std.ArrayList([]const u8) = .empty;
      defer args.deinit(init.gpa);
      while (args_iter.next()) |arg| try args.append(init.gpa, arg);

      app.run(
          init.gpa,
          init.io,
          &stdout.interface,
          &stderr.interface,
          environBlockPtr(init.minimal.environ.block),
          args.items,
      ) catch |err| switch (err) {
          error.OutOfMemory => return err,
          else => std.process.exit(1),
      };
  }

  fn environBlockPtr(block: anytype) [*:null]const ?[*:0]const u8 {
      const T = @TypeOf(block);
      switch (@typeInfo(T)) {
          .pointer => |ptr| {
              if (ptr.size == .slice) return block.ptr;
          },
          .@"struct" => {
              if (@hasField(T, "slice")) return block.slice.ptr;
          },
          else => {},
      }
      return block.ptr;
  }
#+end_src

The =init.minimal.environ.block= representation has drifted across
0.16/0.17 snapshots: some expose =block.slice.ptr=, others =block.ptr=. The
tiny helper above adapts either to the many-item pointer the library wants.
On platforms without a raw environment block (WASI without libc, Windows),
pass any null-terminated =[*:null]const ?[*:0]const u8= you own — env
binding simply finds nothing.

** ⚠️ Error Handling

=App.run= returns =anyerror!void=: handler errors are forwarded verbatim.
Parse-time failures are members of =cli.Error= and are written as
human-readable messages to the supplied =stderr= before any handler runs.

#+begin_src zig
  app.run(...) catch |err| switch (err) {
      error.OutOfMemory => return err,
      else => std.process.exit(1),
  };
#+end_src

#+html:<details>
#+html:<summary><b>The full <code>cli.Error</code> set</b></summary>
#+html:<br>

A caller that wants specific parse exits can match the names:

#+begin_src zig
  app.run(...) catch |err| switch (err) {
      error.OutOfMemory => return err,
      error.UnknownCommand, error.UnknownFlag => std.process.exit(64),
      error.MissingRequiredFlag => std.process.exit(78),
      else => std.process.exit(1), // handler errors and other parse errors
  };
#+end_src

| Error                 | Meaning                                        |
|-----------------------+------------------------------------------------|
| ~UnknownCommand~      | First non-flag token didn't match any command  |
| ~UnknownSubcommand~   | Parent matched but child didn't                |
| ~UnknownFlag~         | ~--foo~ or ~-x~ not declared                   |
| ~MissingFlagValue~    | Value-taking flag had no value                 |
| ~MissingRequiredFlag~ | A ~.required = true~ flag was absent           |
| ~ConflictingFlags~    | Two mutually-exclusive flags set               |
| ~UnexpectedArgument~  | Too many positional arguments                  |
| ~InvalidIntValue~     | ~ctx.flagInt~ got an unparseable value         |

** 📦 Installation

Add this repository to your =build.zig.zon=, then import the module from
=build.zig=. Target Zig =0.17= development for new work; =0.16.0= is the
minimum baseline. The library uses APIs (=std.process.Init=, =std.Io.File=,
=std.Io.Writer.Allocating=) that don't exist on =0.15= or older.

#+begin_src zig
  const dep = b.dependency("zig_cli_kit", .{
      .target = target,
      .optimize = optimize,
  });
  exe.root_module.addImport("zig-cli-kit", dep.module("zig-cli-kit"));
#+end_src

unreleased Zig =0.17= development line (zignix =zig-master=) — the target
version — exposed alongside =zig-0.16= (the 0.16.0 baseline) via zignix's
=withName=. zignix keeps both packages current; the revision is pinned in
=flake.lock=. Bump it with =nix flake update zignix=.

** 🚧 Status

Early extraction. The API is unstable and will change while downstream
tools adopt it. Targets the current Zig =0.17= development line while
preserving =0.16.0= compatibility until 0.17 becomes the baseline.
